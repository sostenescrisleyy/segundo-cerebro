# Checklist de Qualidade Visual

## Antes de Entregar — Verificar

### Tipografia
- [ ] A fonte escolhida é distinta e contextual (não Arial, Inter, Roboto)?
- [ ] O pairing display + body está harmonioso?
- [ ] Hierarquia tipográfica clara (h1 vs h2 vs body visualmente distintos)?
- [ ] Line-height confortável para corpo de texto (1.5–1.7)?
- [ ] Letter-spacing ajustado para headings grandes?

### Cor
- [ ] A paleta é específica para esse contexto (não gradiente roxo genérico)?
- [ ] Contraste WCAG AA para texto (4.5:1 mínimo)?
- [ ] Cores têm propósito semântico definido?
- [ ] Dark mode funciona corretamente se implementado?

### Layout
- [ ] Há um elemento inesperado que quebra a monotonia?
- [ ] O espaçamento é consistente (usando variáveis ou escala)?
- [ ] É responsivo de verdade (testar em 375px, 768px, 1440px)?
- [ ] Não há "SaaS padding" genérico demais?

### Motion
- [ ] Animações respeitam `prefers-reduced-motion`?
- [ ] Nenhuma animação é apenas decoração sem propósito?
- [ ] Transições têm easing natural (não linear)?
- [ ] Staggered reveals criam hierarquia visual?

### Detalhes
- [ ] Nenhum emoji como ícone (usar SVG)?
- [ ] Focus states visíveis e acessíveis?
- [ ] Estados hover são claros e estáveis?
- [ ] Loading states existem onde necessário?
- [ ] Mensagens de erro têm design?

### Geral
- [ ] O design tem um ponto de vista claro?
- [ ] É memorável? O que alguém vai lembrar depois de ver?
- [ ] Combina com a identidade/tom da marca/produto?
